Embedded Quicktime Video in a web page sample

The example above is an embedded Quicktime streaming media clip named “sample.mov” that is stored for streaming on your hosting service. The real location of this file is html/_qt/sample.mov. This is where you should upload all of your videos for streaming. The movie you see uses a reference movie file named "sample-reference.mov" which is located in the directory html/.samples/qt/.

You may also access your media clips directly with the Quicktime Media Player using the following two URLs:

rtsp://ss48.shared.server-system.net:554/jeresmith/sample.mov
(streaming version)

...or

http://jeresmith.com/_qt/sample.mov
(non-streaming version that progressively downloads)

The streaming version actually uses the special DSS (Darwin Streaming Server) software to stream the media clip to the web browser and takes advantage of DSS's special features. The non-streaming version just attempts to play the file directly off of the web server with no special streaming features. Non-streaming media clips can be stored anywhere in your HTML directory.

The HTML code used to embed the media clip above is shown below. You may duplicate this example for any number of other Quicktime videos in your own web pages.

<embed src="sample-ref.mov" width="242" height="199" bgcolor="#666666"
autoplay="true" loop="false" controller="true" pluginspage="http://www.apple.com/quicktime/">
</embed>

Notice that the source "sample-reference.mov" is actually just a small text file sitting in the html/samples/qt/ directory. This is known as a "reference file". It contains one line:

rtsptext rtsp://ss48.shared.server-system.net:554/jeresmith/sample.mov

This is a link to your actual streaming media clip. The reference file method is required when embedding media clips in web pages, and it also makes it possible to have virtual media clips anywhere in your website directory. Of course the real media clips need to be stored in the html/_qt/ directory. Reference movies also permit additional functionality such as loading certain media clips based on the connection speed of your viewing audience.

The sample HTML code above also contains some special options that dictate how the media clip behaves once loaded. (Example: autoplay="true", which automatically plays the clip once the page is loaded.) The are many other variations and options for embedding clips into your web pages. We suggest visiting the knowledge links below to learn more about the special options that can control your embedded clips.
